# Contributing to VividCanvas

Thank you for your interest in contributing to VividCanvas! We welcome contributions from the community. Please follow these guidelines to ensure a smooth collaboration.

## How to Contribute

### 1. Fork the Repository

Fork the [VividCanvas](https://github.com/nayankamal305-hub/VividCanvas) repository to your GitHub account.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/VividCanvas.git
cd VividCanvas
```

### 3. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

Use a descriptive branch name (e.g., `feature/add-auth`, `bugfix/fix-login-issue`).

### 4. Make Your Changes

- Follow the code style and conventions already in the project.
- Write clear, concise commit messages.
- Test your changes before committing.

### 5. Commit Your Changes

```bash
git add .
git commit -m "Type: Brief description of changes"
```

Commit types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`.

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. Open a Pull Request

Navigate to the original repository and click "New Pull Request". Provide:

- A clear title and description of your changes.
- Reference any related issues (e.g., "Closes #123").
- Ensure your code passes all checks.

## Code Standards

- **TypeScript**: Use TypeScript for type safety.
- **Formatting**: Use Prettier for consistent formatting.
- **Linting**: Follow ESLint rules.
- **Testing**: Write tests for new features and bug fixes.

## Reporting Issues

If you find a bug or have a feature request:

1. Check if the issue already exists.
2. Create a new issue with a clear title and description.
3. Include steps to reproduce (for bugs) or a detailed description (for features).
4. Attach screenshots or logs if relevant.

## Code Review Process

1. Your PR will be reviewed by maintainers.
2. Address any feedback or requested changes.
3. Once approved, your PR will be merged.

## License

By contributing, you agree that your contributions will be licensed under the same license as the project (MIT).

## Questions?

Feel free to open an issue or reach out to the maintainers if you have any questions.

Thank you for contributing to VividCanvas!
